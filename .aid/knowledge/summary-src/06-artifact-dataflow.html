6. Artifact Dataflow Across the Pipeline

The "data" AID moves through the pipeline is a sequence of structured markdown files. Each phase produces specific artifacts that downstream phases consume. The Knowledge Base sits behind all of them — updated continuously, referenced everywhere.

The artifact sections (per data-model.md, post-FR2)

Post-FR2 (work-003 feature-002) the per-skill/per-artifact STATE files were consolidated into two area-STATE files — the Discovery-area .aid/knowledge/STATE.md and the per-work .aid/work-NNN-{name}/STATE.md. The Monitor area STATE is deferred until that area matures (per FR2 OQ-3).

  1. Discovery-area STATE.md — Grade, Issues, Q&A, Review History, Calibration Log, Summarization History. Absorbs the retired DISCOVERY-STATE.md + SUMMARY-STATE.md per FR2.
  2. Per-work STATE.md — Features Status, Tasks Status, Deployment, Cross-phase Q&A, Calibration Log. Absorbs retired INTERVIEW-STATE.md + per-feature STATE × N + per-task task-NNN-STATE × N + DEPLOYMENT-STATE.md per FR2.
  3. REQUIREMENTS.md (per work) — feature list + non-functional requirements
  4. SPEC.md (per feature) — requirements + technical spec
  5. PLAN.md — ordered deliveries + Execution Graph (per-task dependencies + parallel groups, per work-001 feature-009)
  6. task-NNN.md — one per task, 8 types (RESEARCH / DESIGN / IMPLEMENT / TEST / DOCUMENT / MIGRATE / REFACTOR / CONFIGURE)
  7. IMPEDIMENT.md — feedback when reality contradicts spec (silent workarounds forbidden; 116-line template at canonical/templates/feedback-artifacts/IMPEDIMENT.md)
  8. known-issues.md — per-work issue tracking (referenced by aid-execute before fix-pass)
  9. Monitor area STATE — DEFERRED per FR2 OQ-3; template pending Q8/H7
  10. project-index.md — generated file inventory (pre-pass for discovery sub-agents; emitted by canonical/templates/scripts/build-project-index.sh)
  11. Recipe templates (work-001 feature-011) — 5 pre-filled lite-path templates at canonical/recipes/ (bug-fix, method-refactor, add-crud-endpoint, add-unit-test, write-release-note) with YAML front-matter + {{slot}} placeholders parsed by parse-recipe.sh
  12. Agent / Skill frontmatter — see api-contracts.md

Every artifact carries a metadata header (Source / Status / Last Updated). The long-lived ones (REQUIREMENTS, SPEC, KB docs) carry a mandatory ## Revision History table — every edit gets a row.
