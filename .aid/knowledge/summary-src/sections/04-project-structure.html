Project Structure

AID is a methodology delivered as a multi-profile CLI installer. The repository has two faces: the product (building and shipping AID) and the dogfood install (using AID to build AID). Version 1.1.1.

Directory / FilePurposeNotes
canonical/Single source of truth for all installable content (skills, agents, templates, scripts, recipes)Never edit profiles/ directly — always edit here
profiles/Five rendered, host-tool-specific install trees (build output)claude-code, codex, cursor, copilot-cli, antigravity
packages/npm/npm package wrapperVendored bin/lib/dashboard; zero runtime deps
packages/pypi/PyPI package wrapperSame content; zero runtime deps
bin/CLI entry point (bin/aid Bash, bin/aid.ps1 PowerShell)Cross-platform CLI
lib/Shared install/update/remove engines (aid-install-core.sh, AidInstallCore.psm1)Polyglot parity: Bash mirrors PowerShell
dashboard/Local web dashboard: server (server.mjs, server.py) + reader (reader/)Read-only, loopback-only, no LLM
tests/82 canonical test suites + bespoke harnesses per languageRun via tests/run-all.sh
site/Astro-based docs site (TypeScript)Published to https://aid.casuloailabs.com
install.shBash curl-bootstrap installerASCII-only, POSIX-compatible
install.ps1PowerShell irm-bootstrap installer5.1-compatible, ASCII-only
.claude/Dogfood install of AID (claude-code profile)Real working install, not example data
.aid/AID pipeline state + Knowledge Base for this repositoryThe dogfood KB you are reading
VERSIONCanonical version string (1.1.1)Must stay in sync with package.json, pyproject.toml, and git tag

Primary languages

Markdown (methodology content), Shell/Bash (installer + scripts), PowerShell 5.1+ (Windows installer), Python ≥3.8 (build tools + dashboard server), JavaScript/Node ≥18 (dashboard server + site), TypeScript (docs site only). Polyglot by design.
