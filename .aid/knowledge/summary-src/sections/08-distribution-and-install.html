Build & Install

Distribution / Install

Single-source canonical directory renders into five byte-identical host-tool install trees via run_generator.py. The persistent aid CLI is installed once per machine (curl/irm bootstrap, or the aid-installer npm or PyPI package), then aid add <tool> copies the selected profile tree into any project; offline installs use aid add <tool> --from-bundle <tar>. (per infrastructure.md, repo-presentation.md)

Generator

run_generator.py + render.py

7-file Python generator. run_generator.py is the entrypoint: loops all 5 profiles/*.toml, calls render_profile (the render.py copy core) once per profile, performs deletion pass via emission-manifest diff, then runs VERIFY (deterministic) (hard) and VERIFY (advisory). render.py is the single copy core: one copy_tree pass over canonical/{agents,skills,aid} — agents and skills receive per-tool frontmatter translation; aid/ is copied verbatim. The 5 per-type renderer functions and 2 emitter self-tests were deleted in work-005 and folded into this single pass. Python 3.11+ stdlib only. (per module-map.md §3)

End-user install

aid CLI

Bootstrapped once per machine: curl … | bash / irm … | iex, or npm install -g aid-installer / pip install aid-installer (PyPI). Per project: aid add <tool> fetches and verifies the release tarball then copies the profile subtree; aid status, aid update [self], aid remove [self]; bare aid shows a project dashboard. Idempotent — identical files are skipped. Offline: aid add <tool> --from-bundle <tar>. Bash 4+ (Linux/macOS/Git Bash) or PowerShell 5.1+ (Windows). (per infrastructure.md)

Uniform layout / 2 agent formats

Per-tool layouts

All 5 install trees share the same {agents,skills,aid} structure under their tool root (.claude/, .codex/, .cursor/, .github/, .agent/). Agent format is uniform markdown for Claude Code, Cursor, Copilot CLI, and Antigravity; Codex uses native TOML (agent_format = "toml", dormant pending E-CODEX-1). Cursor and Copilot CLI remap the Bash tool name (Terminal / shell); MCP-style tools are omitted on hosts that lack them. (per architecture.md, schemas.md §9)

FR12 invariant root AGENTS.md + FR11 protect-on-diff

Four of the five tools — Codex, Cursor, Copilot CLI, Antigravity — write a root AGENTS.md (Claude Code uses CLAUDE.md only). FR12: all four ship a byte-identical root AGENTS.md, enforced by tests/canonical/test-agents-md-invariant.sh as a CI hard gate, so installing a second AGENTS.md-writing tool into a project is no longer a collision. FR11 protect-on-diff: when aid add would overwrite a user-owned root agent file (CLAUDE.md or AGENTS.md) that has been modified since AID wrote it, the incoming version is written as *.aid-new instead of silently overwriting — the user reviews and merges. The former Option-A last-installed-wins handler was removed with setup.sh. (per infrastructure.md §Install Pipeline, architecture.md)

Never edit profile trees directly

Files under profiles/{claude-code,codex,cursor,copilot-cli,antigravity}/ are generator output. Edits are silently overwritten on the next python .claude/skills/generate-profile/scripts/run_generator.py run. Always edit canonical/ and re-run the generator. The emission manifest is the safety boundary — only previously-emitted files are eligible for deletion. (per coding-standards.md §7a)
