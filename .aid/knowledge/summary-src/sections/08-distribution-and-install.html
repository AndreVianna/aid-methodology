Distribution / Install

Single-source canonical directory renders into three byte-identical host-tool install trees via run_generator.py. End users install by running setup.sh or setup.ps1, which copies the selected profile tree into their project. (per infrastructure.md, repo-presentation.md)

Generator

run_generator.py

87-line orchestrator. Loads every profiles/*.toml, calls render_agents, render_skills, render_templates, render_canonical_scripts, render_recipes per profile, performs deletion pass via emission manifest diff, then runs VERIFY (deterministic) (hard) and VERIFY (advisory). Python 3.11+ stdlib only. (per module-map.md §3)

End-user install

setup.sh / setup.ps1

Interactive tool-selection menu (Claude Code / Codex / Cursor). Copies the selected profile tree into the user's project root. Existing identical files are skipped; differing files prompt unless --force is passed. Bash 4+ (Linux/macOS/Git Bash) or PowerShell 5.1+ (Windows). (per infrastructure.md)

Codex split layout

Split-root exception

Codex CLI uses a split layout: profiles/codex/.codex/agents/ holds TOML agent definitions; profiles/codex/.agents/{skills,scripts,recipes,templates}/ holds markdown bodies. A single emission-manifest.jsonl covers both roots. This is the only profile that differs in structure from Claude Code and Cursor. (per project-structure.md §Unusual Structure)

Never edit profile trees directly

Files under profiles/{claude-code,codex,cursor}/ are generator output. Edits are silently overwritten on the next python run_generator.py run. Always edit canonical/ and re-run the generator. The emission manifest is the safety boundary — only previously-emitted files are eligible for deletion. (per coding-standards.md §7a)
