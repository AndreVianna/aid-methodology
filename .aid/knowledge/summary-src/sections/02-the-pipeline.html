The Pipeline

AID defines eight sequential pipeline phases — from brownfield discovery through production monitoring — with explicit human-controlled phase gates and feedback loops. Each phase maps to a slash-command skill. (per architecture.md, repo-presentation.md)

The pipeline is sequential by default: human approval is required to advance between phases. Feedback loops exist as escape hatches — Monitor can route back to Execute or Discover; Execute can issue IMPEDIMENT escalations; any phase can revisit Discovery. The Lite Path shortcut routes small work directly from Interview to task execution, bypassing Specify, Plan, and Detail. (per feature-inventory.md §Engineering features)

Phase gate philosophy

Each skill exits after one state and re-enters on the next slash-command invocation. No auto-advance — the human controls every transition. This is intentional: the human is the Director, not the scheduler. (per architecture.md §Thin-Router state machine)
