Release ledger — release-tracking.md

Release Tracking

A cumulative, newest-first log of every AID release and what shipped in it. Each line is tagged [NEW], [CHANGE], or [FIX]; a top Unreleased section accumulates what is queued for the next version.

Format

Newest-first ledger

Versions are listed top-down, latest first. Read it to know what shipped in which version — and what is still pending release in the Unreleased block.

Item tags

[NEW] / [CHANGE] / [FIX]

[NEW] items lead with a feature name; [CHANGE] and [FIX] are description-only. A trailing version on an Unreleased item marks a planned target.

At tag time

Unreleased → version

When a release is cut, the Unreleased section is renamed to the version + date and a fresh Unreleased block starts. Hand-maintained; the root RELEASE_NOTES.md is retired.
