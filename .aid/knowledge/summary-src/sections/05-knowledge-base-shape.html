Data Model

Knowledge Base Shape

AID has no database. The Knowledge Base is 15 primary Markdown documents under .aid/knowledge/, each with a YAML frontmatter block (kb-category, source, intent, contracts, changelog). The INDEX.md is auto-generated from each doc's intent: field. (per schemas.md)

Fact stability tiers (T1–T4)

Every claim in a KB doc belongs to one of four tiers that govern whether it may appear inline. (per coding-standards.md §9a)

TierWhatInline Allowed?Where It Lives
T1 ConceptPatterns, definitions, architectural lawsYes — this IS the knowledgeInline in primary docs
T2 StructureCardinality contracts, schemas, named-list assertionsYes — when load-bearingInline + contracts: frontmatter
T3 MetricLine counts, byte counts, function countsNo — banned from primary docs.aid/generated/metrics.md only
T4 TemporalDates, cycle tags, "as of X"No — banned from primary docsFrontmatter changelog: + STATE.md history
