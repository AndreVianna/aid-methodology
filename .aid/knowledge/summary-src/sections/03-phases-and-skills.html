Phases & Skills

Ten user-facing slash-command skills, all Shipped status. Each is a Thin-Router state machine (the SKILL.md router is at most ~360 lines) that delegates per-state logic to references/state-*.md files. (per feature-inventory.md, module-map.md)

Setup — Config
/aid-config
View or update AID pipeline settings. Bare invocation shows all values in a table; first run auto-creates .aid/settings.yml from the template. Pass a dotted key to view and update one setting interactively.
Shipped 0 reference files
States
Single-state (no state machine)
Key outputs
.aid/settings.yml
Phase 1 — Discovery
/aid-discover
Brownfield project discovery with built-in quality gate. GENERATE resolves a declared KB doc-set from discovery.doc_set (or synthesizes a default seed), runs a propose→confirm checkpoint, then dispatches discovery sub-agents data-driven from the confirmed set; it then reviews, collects user input, fixes issues, and gets approval.
Shipped 8 reference files
States
GENERATE → REVIEW → Q-AND-A → FIX → APPROVAL → DONE
Key outputs
.aid/knowledge/*.md (the declared/derived KB doc-set)
Agents
5 discovery-* sub-agents + discovery-reviewer
Phase 2 — Interview
/aid-interview
Adaptive requirements gathering through conversational interview. Builds REQUIREMENTS.md incrementally; supports a Lite Path for small work (routes to LITE-BUG-FIX / LITE-DOC / LITE-REFACTOR / LITE-FEATURE sub-paths).
Shipped 19 reference files
States
FIRST-RUN → Q-AND-A → TRIAGE → {full or lite path}
Lite path
CONDENSED-INTAKE → TASK-BREAKDOWN → LITE-REVIEW → LITE-DONE
Agents
interviewer, architect, reviewer
Phase 3 — Specify
/aid-specify
Technical specification through conversational refinement, one feature at a time. Acts as a tech lead — reads KB, Requirements, and codebase, proposes technical solutions, and builds the spec collaboratively with the developer.
Shipped 9 reference files
States
INITIALIZE → CONTINUE → REVIEW → DONE (+ SPIKE / BLOCKED)
Key outputs
SPEC.md per feature
Phase 4 — Plan
/aid-plan
Sequences feature SPECs into deliverables, each one a functional MVP that builds on the previous. Answers: in what order do we deliver, and does each delivery stand on its own?
Shipped 3 reference files
States
FIRST-RUN → REVIEW → DONE
Key outputs
PLAN.md per work area
Phase 5 — Detail
/aid-detail
Breaks deliverables into small, dependency-driven, typed tasks — each one a reviewable unit. Detects task types from SPEC signals and builds an execution graph per delivery with explicit dependencies and parallelism.
Shipped 5 reference files
States
FIRST-RUN → REVIEW → DONE
Task types
RESEARCH / DESIGN / IMPLEMENT / TEST / DOCUMENT / MIGRATE / REFACTOR / CONFIGURE
Phase 6 — Execute
/aid-execute
Implements tasks with a built-in two-tier review loop. Runs a parallel pool (PD-0..PD-6, default MaxConcurrent=5) with BFS failure-block-radius and graceful degradation. Per-task quick-check plus per-delivery quality gate.
Shipped 8 reference files
States
EXECUTE → REVIEW → FIX → DELIVERY-GATE
Key scripts
writeback-state.sh, compute-block-radius.sh, grade.sh
Optional — Deploy
/aid-deploy
Optional, on-demand end-of-pipeline Deliver skill (not a numbered phase). Packages completed deliveries into a release. Selects eligible deliveries, verifies the combined build, packages according to project infrastructure, generates release notes, and updates artifact statuses.
Shipped 5 reference files
States
IDLE → SELECTING → VERIFYING → PACKAGING → DONE
Optional — Monitor
/aid-monitor
Optional, on-demand end-of-pipeline Deliver skill (not a numbered phase; independent of /aid-deploy). Observes production, classifies findings, and routes actions. Combines telemetry interpretation with triage — detects anomalies, performs root cause analysis, and routes bugs to aid-execute or change requests to aid-discover.
Shipped 3 reference files
States
OBSERVE → CLASSIFY → ROUTE → DONE
Optional — KB Viewer
/aid-summarize
Generates a single self-contained knowledge-summary.html from .aid/knowledge/. Inlines Mermaid diagrams for offline use, supports light/dark themes, and enforces a two-grade quality gate (Machine + Human) before writing the final output.
Shipped 10 reference files
States
PREFLIGHT → STALE-CHECK → PROFILE → GENERATE → VALIDATE → MANUAL-CHECKLIST → FIX → APPROVAL → WRITEBACK → DONE
