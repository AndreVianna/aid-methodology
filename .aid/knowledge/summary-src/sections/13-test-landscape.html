installer-tests.yml Push to any feature branch Installer integration tests; Linux + Windows PS 5.1 lane; runs tests/windows/Test-AidInstaller.ps1 on windows-latest
release.yml Git tag push (v*) Full release pipeline: build, sign, publish npm + PyPI, create GitHub release
docs.yml Push to master only Astro docs site build and deploy to GitHub Pages

Critical: tests must pin HOME

The migration scanner defaults its root to $HOME. Any test that fires it must export HOME=<throwaway> (not just AID_HOME) and include an escape canary, or it will migrate the developer's real repos. This has caused issues 3 times during development.

Known Test Gaps

Gap

No coverage % metric

Test coverage is assessed by presence/absence of suites, not line percentage. A coverage metric tool is tech debt L4.

Gap

Heavy gates master-only

The canonical suite and docs build run only on master CI. Feature branches skip them; run locally before claiming green.

Gap

Windows tests outside run-all

The Windows PowerShell 5.1 installer tests (Test-AidInstaller.ps1) run only on windows-latest CI; they are not included in run-all.sh.
