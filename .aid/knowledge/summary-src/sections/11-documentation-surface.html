Documentation Surface

The AID GitHub repository is itself a documentation product. The user-facing surface has five layers: README (pitch and entry point), docs/ (reference), examples/ (case studies), methodology spec (load-bearing artifact), and install scripts. (per repo-presentation.md)

README.md — 388 lines

Primary pitch document

Eight sections with progressive disclosure: What is AID, Why AID (failure modes it removes), The Pipeline (Mermaid flowchart + 10 skills), The Knowledge Base (14-doc fixed shape), The Agent Model (3-tier diagram), Feedback Loops (11 formal loops), AID vs. SDD (8-row comparison), Using AID in your project (install instructions). (per repo-presentation.md)

methodology/aid-methodology.md — 1,070 lines

Load-bearing specification

The complete methodology specification, version 3.1. Every skill and agent is derived from this document. It is read by humans, not by skills directly. It defines the phase model, feedback loops, phase gates, the human-AI Director/Orchestrator/Specialist roles, and the work-type taxonomy. (per project-structure.md §Methodology spec)

AID applied to itself

This repository dogfoods AID: the top-level .claude/ tree is functionally identical to profiles/claude-code/.claude/ (the install bundle) but is the one the development environment actually uses. This means every AID skill has been exercised against this very codebase — discovery, interview, specify, plan, detail, execute, deploy, and summarize have all been run here.
