Docs

Documentation Surface

The AID GitHub repository is itself a documentation product. The user-facing surface has four layers: README (lean adopter-first entry point), docs/ (reference — spec, install guide, release guide, FAQ, glossary, repo structure), examples/ (case studies), and the methodology spec (docs/aid-methodology.md, v3.2, moved from the former methodology/ directory). (per repo-presentation.md)

README.md

Primary pitch document

Lean, adopter-first entry point: What is AID, Why AID (failure modes it removes), The Pipeline (Mermaid flowchart + 11 skills), The Knowledge Base (declared/derived doc-set), The Agent Model (3-tier diagram), Feedback Loops (11 formal loops), AID vs. SDD (8-row comparison), Using AID in your project (install via aid add). (per repo-presentation.md)

docs/aid-methodology.md

Load-bearing specification

The complete methodology specification, version 3.2. Every skill and agent is derived from this document. It is read by humans, not by skills directly. It defines the phase model, feedback loops, phase gates, the human-AI Director/Orchestrator/Specialist roles, and the work-type taxonomy. (per project-structure.md §Methodology spec)

docs/ folder

Reference material

Six files: docs/aid-methodology.md (load-bearing spec, v3.2), docs/repository-structure.md (repo layout guide), docs/install.md (installer reference), docs/release.md (release process), docs/faq.md (frequently asked questions), docs/glossary.md (term definitions). The methodology spec moved here from the former methodology/ directory. (per project-structure.md §Documentation)

examples/ folder

Case studies (3)

Three real-world proof-of-concept case studies: brownfield-enterprise/ (discovery report + architecture KB), data-pipeline/ (multi-agent pipeline architecture), desktop-app/ (greenfield delivery plan + task spec). Note: the two March-2026 examples (data-pipeline, desktop-app) are ~2 months old; brownfield-enterprise was refreshed 2026-05-22 — none is 3+ months stale (see tech-debt L5). (per project-structure.md §Examples)

AID applied to itself

This repository dogfoods AID: the top-level hand-maintained .claude/ tree carries the same bodies as the profiles/claude-code/ install bundle but is the one the development environment actually uses. This means every AID skill has been exercised against this very codebase — discovery, interview, specify, plan, detail, execute, deploy, and summarize have all been run here. The dogfood install (internal tooling, not part of the methodology) is detailed in §14 Dogfooding (internal). (per project-structure.md §Unusual Structure)
