Agent Model & Tiers

9 specialist agents across three model tiers (Large / Medium / Small). Skills dispatch agents via the host tool's Agent capability. The key invariant: the reviewer's tier is always greater than or equal to the executor's, so a writer never grades its own work. (per architecture.md, module-map.md)

Large Tier — 4 agents

Heavy-lifting analysts

aid-interviewer, aid-architect, aid-researcher, aid-reviewer. The Researcher consolidates the five former discovery cataloguing agents and the analysis surfaces of security and performance. The Reviewer consolidates the former KB-document review role. (per module-map.md §2a)

Medium Tier — 4 agents

Specialists + executors

aid-developer (the ONLY agent authorized to modify production code), aid-operator, aid-orchestrator, aid-tech-writer. The Developer consolidates the former migration-engineering and infra-configure-executor roles. (per module-map.md §2b)

Small Tier — 1 agent

Mechanical utilities

aid-clerk. Labelled INTERNAL UTILITY — sub-agent only, do NOT invoke from a skill. Consolidates the three former small-tier utility agents (one per operation type: extract, format-fill, glob). Used for mechanical extraction, template filling, and glob enumeration. (per module-map.md §2c)

Reviewer-tier invariant

The reviewer is always at a tier level greater than or equal to the executor. This means the agent that built an artifact is never the same model that reviews it — a structural safeguard against self-grading. (per architecture.md §Three-tier agent dispatch)
