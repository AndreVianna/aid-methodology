Agent Model & Tiers

22 specialist agents across three model tiers (Large / Medium / Small). Skills dispatch agents via the host tool's Agent capability. The key invariant: the reviewer's tier is always greater than or equal to the executor's, so a writer never grades its own work. (per architecture.md, module-map.md)

Large Tier — 10 agents

Heavy-lifting analysts

Architect, reviewer, interviewer, security, and the five discovery sub-agents (scout, architect, analyst, integrator, quality) plus discovery-reviewer. Discovery sub-agents share permissionMode: bypassPermissions and background: true. (per module-map.md §2a)

Medium Tier — 9 agents

Specialists + executors

Orchestrator, researcher, developer (the ONLY agent authorized to modify production code), operator, data-engineer, performance, devops, tech-writer, ux-designer. The developer agent is the sole production-code writer by explicit contract. (per module-map.md §2b)

Small Tier — 3 agents

Mechanical utilities

simple-extractor, simple-formatter, simple-glob. Labelled INTERNAL UTILITY — sub-agent only, do NOT invoke from a skill. Used for mechanical extraction, template filling, and glob enumeration. (per module-map.md §2c)

Reviewer-tier invariant

The reviewer is always at a tier level greater than or equal to the executor. This means the agent that built an artifact is never the same model that reviews it — a structural safeguard against self-grading. (per architecture.md §Three-tier agent dispatch)
