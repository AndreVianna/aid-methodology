File-format contracts

settings.yml

Runtime configuration

YAML 1.2, 5 mandatory top-level sections: project, tools, review, execution, traceability. Per-skill minimum_grade overrides fall back to global review.minimum_grade then the hardcoded default (A). (per schemas.md Â§2)

emission-manifest.jsonl

Deletion-safety boundary

JSON Lines, LF-only. Sentinel first line {"_manifest_version": 1}. Four-key record schema: profile, src, dst, sha256. Sorted by dst for byte-stable diffs. Only files in the manifest are eligible for deletion on the next render pass. (per schemas.md)

STATE.md

Per-area state hub

One STATE.md per area (.aid/knowledge/STATE.md for Discovery; .aid/{work}/STATE.md per work area). Absorbs legacy per-feature and per-task STATE files (FR2 area-STATE consolidation). Sections: External Documentation, KB Documents Status, Q&A, Review History, Summarization History. (per schemas.md Â§3)

Heartbeat files

L3 subagent visibility

Single-line files at .aid/.heartbeat/{agent}-{unix-ts}.txt. Format: [YYYY-MM-DDTHH:MM:SSZ] STATE | progress | activity (~eta). Written with > (overwrite), never >>. Activity must change between writes. (per coding-standards.md Â§5a)
