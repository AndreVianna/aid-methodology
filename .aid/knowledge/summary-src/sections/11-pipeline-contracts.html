Pipeline Contracts

The typed artifact contracts and state-machine transitions between AID pipeline phases — what each phase consumes, produces, and gates on. Read this before changing any skill, artifact template, or phase hand-off.

Config (bootstrap) /aid-config cwd (repo root) .aid/settings.yml, KB STATE.md scaffold Human: settings confirmed

The Eleven Feedback Loops

Any phase can revise upstream artifacts through a named feedback loop, producing a traceable record rather than a silent rewrite. AID defines 11 named loops (abbreviated names from pipeline-contracts.md):

L1

Discover → KB Update

Discovery findings require updating an existing KB document.

L2

Interview → Discover

Q&A reveals a KB gap; targeted update triggered before spec.

L3

Specify → Requirements

Specification reveals an ambiguity or missing requirement.

L4

Plan → Specify

Planning reveals an under-specified feature; spec revised.

L5

Detail → Plan

Task breakdown reveals delivery sequencing needs adjustment.

L6

Execute → Detail

Implementation reveals a task that needs re-scoping or splitting.

L7

Execute → Specify

Spec assumption proven wrong during implementation.

L8

Execute → KB

Implementation discovers new architectural knowledge for the KB.

L9

Review → Execute

Reviewer finds issues; executor fixes before delivery gate passes.

L10

Deploy → Execute

Release verification fails; delivery sent back for fixes.

L11

Monitor → Interview

Production finding triggers a new Work via a new interview.
