**Scope:**
- Add automated tests exercising the full list-management round-trips end to end (op envelope ->
  feature-001 dispatch -> writer -> disk -> re-read), covering both registries. No production code is
  written by this task; it validates the behaviors tasks 018-022 implement.
- **connector.set / connector.remove round-trips** -- assert a `connector.set` op authors exactly one
  `<root>/<stem>.md` (atomic single-entry, no sibling descriptor touched), the stem matches the
  skill slug rule, per-type normalize holds (`mcp` -> `auth_method: none` + no `secret_reference`;
  `ssh` -> `ssh-key`; `api`/`url`/`cli` require `endpoint`+`auth`), the `secret_reference` default
  (`file:.aid/connectors/.secrets/<stem>`) is applied for a credentialed connector with `secret_ref`
  omitted, and no secret VALUE is ever written; `connector.remove` purges the secret, deletes the
  descriptor, and is a clean no-op for an absent stem. Assert `INDEX.md` is regenerated by
  `build-connectors-index.sh` and is byte-identical across two runs over an identical descriptor set
  (determinism/idempotence). (feature-007 AC1, AC2)
- **external-source.add / external-source.remove round-trips** -- assert `add` inserts one contiguous
  `  - <value>` item under `sources:` (dropping a lone `- (none)` placeholder), mirrors a `- <value>`
  bullet into the managed `## Sources` block, is an exit-0 no-op for an already-present value, and
  byte-preserves every non-target line; `remove` deletes the matching frontmatter item (writing
  `sources: []` when the real list empties) and the body bullet (restoring the canonical paragraph
  when it empties), and exits 1 (->404) for an absent value. Assert an added entry keeps
  `lint-frontmatter.sh` green (its alphabet matches `sources_entry_shape()`). (feature-010 AC1, AC2)
- **exit -> HTTP mapping** -- assert the endpoint maps each writer exit code to the correct HTTP
  status via feature-001's map with no per-op remapping: connectors `0`->200, `4`/`5`->422,
  `3`|`6`->500 (never 404/409 -- the writer never emits `1`/`2`); external sources `0`->200,
  `1`->404, `2`->409, `3`->500, `4`->422. Assert per-op arg-schema violations return 422 BEFORE any
  spawn, and the `write_enabled` gate returns 403 under read-only.
- **parser twin parity** -- assert `parse_connectors` / `parseConnectors` produce byte-identical
  output over a shared connectors fixture set (enumeration filter, sort-by-stem, six-scalar
  extraction, missing-dir -> `[]`), and `parse_external_sources` / `parseExternalSources` produce
  byte-identical output over a shared `external-sources.md` fixture set (`(none)` filtered, order
  preserved, dedupe, absent-file -> `[]`), following the existing cross-runtime parity harness
  pattern (`node --input-type=module` against `reader.mjs` byte-compared to the Python reader; e.g.
  `test_task044_freshness_parity.py`). Assert the DM-1 serializer fixtures for both twins
  (`test_server_py.py` / `test_server_node.mjs`) stay byte-consistent with `connectors` +
  `external_sources` present and `schema_version` unchanged.
- **fixtures** -- add the shared golden fixtures the round-trip and parity assertions consume
  (connector descriptors across all five types incl. credentialed + `mcp`; an `external-sources.md`
  with placeholder-only, single-entry, and multi-entry states), placed under the existing
  `dashboard/reader/tests/` and `dashboard/server/tests/` fixture locations.
