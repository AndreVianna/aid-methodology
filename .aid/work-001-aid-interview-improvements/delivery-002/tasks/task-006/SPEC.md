# task-006: T4 multi-viewport overflow-clip check in validate-visuals.mjs (M4 impl)

**Type:** IMPLEMENT

**Scope:**
- Add the **T4 "no horizontal overflow-clip at target widths"** check to the canonical
  source-of-truth **`canonical/aid/scripts/summarize/validate-visuals.mjs`** (NOT a profile copy --
  the `.mjs` is generator-emitted; see the regen AC below). It currently renders each visual at the
  single default viewport (no `setViewport`, ~1280px) and runs T1/T2/T3 only.
- **Target widths:** a named constant array `OVERFLOW_VIEWPORTS = [732, 390]` (dashboard column
  ~732px + mobile ~390px), placed alongside the existing `MIN_FONT_SIZE_PX` / `OVERLAP_TOLERANCE`
  constants so they are tunable. For each width, `page.setViewport({ width, height })` (or a fresh
  page per width) and re-evaluate each visual.
- **Container-relative predicate (the one real M4 design risk feature-007 assigned to Detail --
  RESOLVE it here, do not leave open):** T4 measures overflow relative to the visual's OWN responsive
  container, NOT the raw viewport, so that fixed-pixel-width `kb.html` SVGs do not make T4 a
  false-positive generator. Implement the predicate as `el.scrollWidth > el.clientWidth + EPS`
  (content wider than its own container) -- and only fall back to the bounding-rect-vs-nearest-
  layout-container test where `clientWidth` is unreliable. Use a small px `EPS` to avoid sub-pixel
  false positives. The contract: "a visual horizontally clipped WITHIN ITS CONTAINER fails at a
  target width"; a fixed-width SVG inside a width-constrained, scroll-free container does NOT fail.
- **Plug-in points (per feature-007 M4):** add T4 to the per-visual evaluation loop alongside
  T1/T2/T3; emit a `T4 overflow-clip (widths 732/390): PASS|FAIL` line per visual; fold T4 into the
  per-visual pass/fail and the final `passCount` gate; add a T4 entry to the failure-explanation
  block and the `--check-only` summary. Do not break T1/T2/T3.
- IMPLEMENT scope = the validator code + the run_generator.py regen of the emitted copies. The
  behavioral proof that T4 catches a real clip (Playwright-rendered) and the test-suite wiring is
  task-007 (TEST).
