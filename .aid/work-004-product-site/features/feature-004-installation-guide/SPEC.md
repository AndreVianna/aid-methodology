## Description

The full installation how-to under Guides, sourced from `docs/install.md`. It covers a
channels overview and per-channel instructions for all four channels — curl/irm, npm, PyPI,
and the offline tarball — plus per-tool "add to project" instructions (Claude Code / Codex /
Cursor / Copilot CLI / Antigravity) in tabbed blocks, and update / remove procedures. Every install command
renders the latest released version injected at build time (feature-008), so the documented
commands never go stale.

## User Stories

- As a new adopter, I want per-channel install instructions for curl/irm, npm, PyPI, and offline so that I can install via whichever channel fits my environment.
- As a new adopter, I want per-tool "add to project" instructions in tabbed blocks so that I can wire AID into my specific agent tool quickly.
- As a new adopter in an air-gapped environment, I want the offline tarball install documented so that I can install without internet access.
- As a returning user, I want update and remove instructions so that I can manage an existing install.
- As a returning user, I want install commands to always show the current version so that copied commands work without edits.

## Technical Specification

> provider contracts of feature-001 (site foundation) and feature-008 (version injection).

### Overview & Approach

This feature ships **one hand-authored MDX page** — `site/src/content/docs/guides/installation.mdx`
— that is the canonical Installation guide under **Guides**. It is editorially structured for the
website (channels overview → per-channel install with per-OS tabs → per-tool "add to project" tabs
→ update/remove), drawing its prose and flows faithfully from `docs/install.md` (AC5) while sourcing
**every version-bearing command from feature-008** so commands never go stale (AC13).

It extends feature-001's `site/` Astro + Starlight project (do not contradict its anchors):
content lives under `site/src/content/docs/`, the page occupies the `guides/installation` slug that
feature-001's sidebar already reserves for feature-004, and it uses Starlight's stock content
components (`<Tabs>`/`<TabItem>`, `<Steps>`, `<Aside>`, `<LinkButton>`). It consumes feature-008's
`<InstallCommand channel="curl|irm|npm|pypi|offline" />` for **every** install one-liner — no version is
ever hard-coded (D5 below).

### Architectural Decisions

| # | Decision | Rationale |

### Page Structure — `site/src/content/docs/guides/installation.mdx`

Frontmatter:

```yaml
---
title: Installation
description: Install, update, and remove AID across curl/irm, npm, PyPI, and offline bundle channels — with per-OS and per-tool instructions.
sidebar:
  order: 1
---
import { Tabs, TabItem, Steps, Aside, LinkButton } from '@astrojs/starlight/components';
import InstallCommand from '../../../components/InstallCommand.astro';
```

(Import depth `../../../components/` = from `content/docs/guides/` up to `src/`, per feature-008's
documented consumer import; spot-check against the built path — see A2.)

Section outline (H2 unless noted):

1. **How it works (two layers)** — the two-step model: the global `aid` CLI (bootstrap once per
   machine into `~/.aid` / `%LOCALAPPDATA%\aid`) vs. per-project files added by `aid add <tool>`
   (manifest-tracked). Sourced from `docs/install.md` "How it works (two steps)".
2. **Choose your install channel** (D4) — comparison table (Channel · Requires · When to use) for
   curl/irm, npm, PyPI, offline, plus a one-line "pick one channel per machine" note
   (`docs/install.md` "Install channels" + "Multiple channels on PATH").
3. **Step 1 — Bootstrap the `aid` CLI** — per-channel install, each channel's command rendered via
   `<InstallCommand>` and wrapped in **per-OS** `<Tabs syncKey="os">`:
   - **curl / irm** — `<TabItem label="Linux">`/`"macOS"` → `<InstallCommand channel="curl" />`
     (Linux/macOS bash one-liner); `<TabItem label="Windows">` → `<InstallCommand channel="irm" />`,
     which renders the version-pinned Windows PowerShell `irm … | iex` bootstrap (powershell-highlighted).
     feature-008 exports `curl` and `irm` as distinct channels, so **every** OS tab here is
     version-injected by the component — no fenced fallback (see A1, resolved).
   - **npm** — `<InstallCommand channel="npm" />` (OS-agnostic; one tab note that it is identical
     on all OSes). Requires Node >=18. Includes `npx aid-installer add <tool>` no-global variant
     and `aid update self` → `npm install -g aid-installer@latest`.
   - **PyPI** — `<InstallCommand channel="pypi" />` (pipx recommended) + `pip install --user`
     fallback; `aid update self` → `pipx upgrade aid-installer`. Requires Python >=3.8.
   - **offline / `--from-bundle`** (D6) — `<Steps>`: download the per-tool tarball +
     `SHA256SUMS`, verify (Linux `sha256sum`, macOS `shasum -a 256`, Windows PowerShell
     `Get-FileHash`) in `<Tabs syncKey="os">`, then `aid add <tool> --from-bundle …`. Documents the
     `aid-<tool>-v<version>.tar.gz` naming and the directory form for multiple tools; links to the
     Releases page (feature-009) for the live asset list.
4. **Step 2 — Add AID to your project (`aid add <tool>`)** — **per-tool** `<Tabs syncKey="tool">`
   with five `<TabItem>`s (labels: Claude Code, Codex, Cursor, Copilot CLI, Antigravity). Each tab:
   the canonical `aid add <id>` command (fenced, D5) + a "what gets installed" summary
   (e.g. claude-code → `.claude/skills/`, `.claude/agents/`, `CLAUDE.md`), grounded in
   `docs/install.md` "What gets installed per tool". Followed by shared prose: `aid status`,
   multi-tool (`aid add codex,cursor`), `--version` pin, auto-detect note, and an `<Aside>` on
   protect-on-diff for root agent files (`CLAUDE.md`/`AGENTS.md`).
5. **One-line first install** — bootstrap + add in one command: Linux/macOS
   `… | bash -s -- add claude-code`; Windows `$env:AID_TOOL = 'claude-code'; irm … | iex`
   (`<Tabs syncKey="os">`). Sourced from "One-line first install".
6. **Update** — `aid update`, `aid update <tool>`, `aid update --version`, and `aid update self`
   (channel-aware table) — fenced blocks (no pinned version to inject). Sourced from "Step 2 —
   Update installed tools / Update the aid CLI itself".
7. **Remove** — `aid remove <tool>`, `aid remove` (confirm/`--force`), `aid remove self`, and the
   manifest-driven safety note. Sourced from "Removing AID" + "Version recording and the manifest".
8. **Next steps / see also** — `<LinkButton>`s to Get Started → Your first work (feature-003) and
   to the CLI **Reference** (feature-006) for the full flag/exit-code/env-var surface (D7).
