# Feature Inventory

> Source: user-provided list enriched with codebase analysis.

> **Status value convention** (used by `build-metrics.sh` for the shipped/partial tally):
> Use one of `✅ Shipped`, `⚠️ Partial`, `❌ Missing`, or `📋 Planned` in the Status
> column. The metrics builder counts `✅ Shipped` and `⚠️ Partial` occurrences. Other
> conventions (e.g., plain `Shipped` text without emoji) are accepted but will not
> appear in the auto-tally — adjust `build-metrics.sh` if your project uses a different
> convention.
