## The pipeline at a glance

**human gate** â€” you approve before the next phase begins. The flow starts at configuration and ends
at delivered, optionally monitored, code.

*Three entry points Â· 82 skills â€” 14 classic pipeline/on-demand skills, the `/aid-triage`
router, and 67 verb-first shortcut skills. Name your change with a **shortcut** and the
shortcut engine collapses Describe through Detail into one autonomous run straight to
Execute; ask **`/aid-triage`** when you're not sure which fits; or run **`/aid-describe`**
for broad or new-project work through the full six-phase pipeline.*

The six core phases, in order (full path):

1. **Discover** â€” map an existing codebase into a Knowledge Base the agents can reason over. (Brownfield only; greenfield starts at Describe (Phase 2a).)
2. **Describe â†’ Define** â€” gather requirements conversationally, then decompose them into features.
3. **Specify** â€” turn each feature into a technical SPEC, with an agent acting as tech lead.
4. **Plan** â€” sequence SPECs into deliveries, each a functional, buildable MVP.
5. **Detail** â€” break deliveries into small, typed, dependency-ordered tasks.
6. **Execute** â€” run each task with a built-in adversarial review loop until it clears the grade bar.

82 skills deliver AID: the 14 classic pipeline/on-demand skills above, the `/aid-triage`
suggest-only router, and 67 verb-first shortcut skills (`/aid-fix`, `/aid-create-api`, â€¦).
When you already know your change, a shortcut skips straight to Execute â€” the same rigour,
proportionate overhead.

## Install

AID installs as a global `aid` CLI; you then run `aid add <tool>` inside a repository to drop the
skills, agents, and Knowledge Base scaffold into your project. Pick the channel for your platform:
