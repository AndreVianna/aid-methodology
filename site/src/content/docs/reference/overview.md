The Reference section is information-oriented: exact facts, tables, and definitions you look up rather than read linearly. Use this section when you already understand AID and need a specific answer quickly.

## What is in this section

| Page | What it covers |
|------|----------------|
| [CLI & subcommands](/reference/cli/) | Every `aid` command and flag — `add`, `update`, `remove`, `status`, `version`, `self`; exit codes; environment variables; tool auto-detect |
| [Skills](/reference/skills/) | All 11 AID pipeline skills — what each skill does and its source definition in `canonical/` |
| [Agents](/reference/agents/) | All 9 AID pipeline agents — role, tier, tools allowed, and source definition |
| [Knowledge Base doc types](/reference/kb/) | The 14 standard KB document templates and what each captures |
| [Settings keys](/reference/settings/) | Every `.aid/settings.yml` key — path, current value, and description |
| [Artifacts](/reference/artifacts/) | Every named file the AID methodology produces — what it is and where it lives |
| [Repository structure](/reference/repository-structure/) | How the AID repository is laid out — `canonical/`, `profiles/`, `docs/`, `bin/`, and more |
| [Glossary](/reference/glossary/) | Definitions of AID terms, phases, artifact names, and install terms |

## Looking for how-to content?

The Reference section deliberately contains no tutorials or procedural steps. For installation instructions see [Installation guide](/guides/installation/), and for pipeline execution see [Working the pipeline](/guides/pipeline/).

For conceptual background — what AID is and why it works the way it does — see the [Concepts](/concepts/overview/) section.
