- `[CRITICAL]` — Wrong information, missing critical sections, would cause bad decisions; security vulnerabilities; build failure.
- `[HIGH]` — Significant gaps, shallow coverage of important areas, missing test coverage on critical paths.
- `[MEDIUM]` — Missing depth in an important area; incomplete but not wrong.

- `[MINOR]` — Cosmetic, formatting, stylistic, nice-to-have improvements.

- `[CODE]` — Pure code quality: readability, duplication, dead code.
- `[TASK]` — Doesn't meet acceptance criteria from `task-NNN.md`.

- `[KB]` — Deviates from coding standards in `.aid/knowledge/coding-standards.md`.
