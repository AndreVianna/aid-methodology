## Review process (reviewer)

### Step 1: Load context

Read:
- Work-root `SPEC.md` â€” for `## Goal`, `## Context`, `## Acceptance Criteria`,
  `## Tasks`, `## Execution Graph`
- All `tasks/task-NNN.md` files
- `STATE.md ## Triage` â€” for `Sub-path`
- `../../../templates/grading-rubric.md` â€” the universal grading rubric
- `.aid/knowledge/INDEX.md` â€” for KB context if relevant

### Step 2: Write findings to ledger and grade

Evaluate the task set against the work-root `SPEC.md` using the universal rubric.
For the lite path, the key checks are:

| Check | Criterion |
|-------|-----------|
| Coherent breakdown | Every task is a single reviewable unit; types are not mixed |
| Traceability | Every task traces to at least one `## Acceptance Criteria` entry |
| Testable criteria | Each task's AC is concrete and verifiable (not vague) |
| Graph completeness | No gaps in the Execution Graph; every task with dependencies lists them |
| Sub-path contract | SPEC.md shape matches the sub-path's required shape (see CONDENSED-INTAKE) |
| Scope fit | Task count is within the sub-path's expected range (1â€“2 for BUG-FIX, 1 for DOC, 1â€“3 for REFACTOR, 1â€“5 for FEATURE) |

Write findings to `.aid/.temp/review-pending/interview-<work>-lite.md` per
`.claude/templates/reviewer-ledger-schema.md`. The table is the entire file content
â€” no headers, no narrative. Each issue is one row with Severity and Status: Pending.

The grade is computed by the orchestrator via grade.sh (not assigned by the reviewer
directly).

**Minimum grade:** read from `bash .claude/scripts/config/read-setting.sh --skill interview --key minimum_grade --default A` if it

- **Grade >= minimum (or [1] override):**
  - Add lifecycle entry: `| {today} | LITE-REVIEW complete â€” Grade: {grade} | /aid-interview LITE-REVIEW |`

- **[3] Loopback to TASK-BREAKDOWN:**
  - Delete the `tasks/` folder contents (task files are regenerated by L2).
  - Reset `## Tasks` and `## Execution Graph` in SPEC.md to placeholder rows.
  - Reset `STATE.md ## Tasks Status` to empty.
  - Add lifecycle entry: `| {today} | LITE-REVIEW loopback â†’ TASK-BREAKDOWN â€” {reason} | /aid-interview LITE-REVIEW |`
